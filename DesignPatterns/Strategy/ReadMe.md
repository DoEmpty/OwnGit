**策略设计模式

+ 定义
	封装一组算法，每个算法为独立的类，可以相互替代，

+ 需求
> 公司有研发，测试，产品三个岗位，每个岗位包括P1,P2,P3,P4,P5,P6,P7共七个层级
	P1层级的员工薪水仅包括基础工资、绩效工资、全勤奖金，不同岗位薪水不同在于P1层级的员工基础薪水不同
	P2-P7层级的员工具有不同的薪水倍率，构成为（基础工资+绩效工资）* 倍率 + 全勤 + 其他，每个层级的其他部分可能都不一样
	倍率如下：P2:1.2,P3:1.6,P4:1.9,P5:2.5,P6:3.4,P7:5.0

+ 实现
>  使用策略模式构建一个LevelSalary基类，基类中通过构造函数注入一个JobCategory(岗位类别)的实例，不同层级的实现类实现不同层级的薪水计算策略
	构建一个策略模式需要的上下文(SalaryContext)类，该类中通过工厂模式对用户输入进行解析，构造正确的LevelSalary实例，并调用LevelSalary的薪水计算方法返回正确的薪水
	客户端提供层级以及岗位名称，调用SalaryContext类的获取薪水方法获取结果。
